Уважаемые коллеги!

Просим Вас обратить внимание на следующие особенности версий подсистемы HRS_RT.

Начиная с версии 086.00 в подсистеме HRS_RT реализован ФК 4LEVELRATE, в рамках которого возможно определение роумингового оператора (rmop_id) для локальных вызовов.

Поиск трафика в функции traffic_searching с учетом rmop_id происходит по следующему алгоритму:

Для версий HRS_RT с 086.00 по 088.02

- если rmop_id не определился (rmop_rmop_id = null) в таблице TRAFIC_HISTORIES производится поиск записи с rmop_rmop_id = null.

- если rmop_id определился, то в таблице TRAFIC_HISTORIES производится поиск записи с rmop_rmop_id равным данному значению. Если такая запись не будет найдена, то будет выполнен поиск записи с rmop_rmop_id = null.

Для версий HRS_RT до 086.00 и начиная с 088.03

- если rmop_id не определился (rmop_rmop_id = null), то считается, что вызов локальный и по умолчанию получает специальное значение – 0 (свой абонент в своей сети). При этом в таблице TRAFIC_HISTORIES производится поиск записи с rmop_rmop_id = 0. Если такая запись не будет найдена, то будет выполнен поиск записи с rmop_rmop_id = null.

- если rmop_id определился, то в таблице TRAFIC_HISTORIES производится поиск записи с rmop_rmop_id равным данному значению. Если такая запись не будет найдена, то будет выполнен поиск записи с rmop_rmop_id = null.

Начиная с версии HRS_RT  086.00 если rmop_id (сеть посещения) совпадает с собственной сетью абонента, то перед поиском записи с rmop_rmop_id = null будет выполнен поиск записи с rmop_rmop_id = 0, то есть потенциально traffic_searching может выполнить 3 поиска. Таким образом если НСИ соответствующим образом заполнена, то будет определяться собственная сеть абонента и traffic_searching должен находить запись с rmop_rmop_id = 0.

При этом, начиная с HRS_RT  086.00, возможна отличная от предыдущих версий тарификация (т.к. тариф из таблицы TRAFIC_HISTORIES будет браться с учетом найденного rmop_id для локального вызова).

Примеры проблемных ситуаций

1. Имеются записи

select count(*) from trafic_histories th

where sysdate between th.start_date and th.end_date

and th.rmop_rmop_id = 0;

и по таблице ROAM_GT_RANGES определяется rmop_id (сеть посещения)  для собственных абонентов, которая не совпадает с собственной сетью абонента. Для исправления необходима корректировка НСИ (за конкретными рекомендациями обращаться в Отдел Технического Сопровождения).

2. В таблице ROAM_GT_RANGES ошибочно заданы собственные диапазоны GT, в результате чего некорректно определяется rmop_id для собственных абонентов. Для исправления необходимо правильно задать диапазоны GT.
